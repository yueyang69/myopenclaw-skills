# Dual-Expert Chat Skill - 双专家对话技能

## 描述
本技能提供一个灵活的双模型对话框架，支持用户以两种模式进行深度分析和决策。**请注意，本技能仅支持通过显式命令触发，不会根据对话内容自动启动，以避免不必要的 Token 消耗。**

1. **大脑 + 手模式 (Brain+Hand Mode)**：一个能力更强的模型（例如 Claude Sonnet）负责规划、决策和高级分析，而一个成本更低的模型（例如 Qwen）负责执行具体的任务、生成代码或提供初步反馈。
2. **辩论 + 协作模式 (Debate+Collaborate Mode)**：两个模型从不同角度对同一问题进行分析、辩论和协作，以提供更全面的视角和更严谨的解决方案。

### 🔄 降级策略（Fallback）

技能自动检测运行环境，按以下优先级执行：

| 优先级 | 模式 | 说明 | 适用场景 |
|--------|------|------|----------|
| 1 | **子代理模式** | 启动两个独立子代理，不同模型并行执行 | Discord/钉钉等支持 `thread=true` 的渠道 |
| 2 | **会话切换模式** | 当前会话依次调用不同模型 | Webchat 等不支持持久会话的渠道 |

**自动降级触发条件：**
- `thread=true` 不可用（渠道未注册 subagent_spawning hooks）
- 子代理启动超时
- 模型调用失败

降级时会自动通知用户：`⚠️ 子代理模式不可用，降级到当前会话切换模型模式`

## 功能
- **模型配置**：可指定两个模型，一个为主导（如规划），一个为辅助（如执行或另一视角）。
- **多种对话模式**：
    - **模式 A (大脑 + 手)**：一个模型负责高阶思考，另一个模型负责低阶执行。
    - **模式 B (辩论 + 协作)**：两个模型扮演不同专家角色，相互补充或辩论。
- **流程编排**：支持如"讨论 → 执行 → 检查 → 总结 → 评分"等多步流程。
- **人工介入点**：可配置在每一步骤之间等待用户确认，或在用户要求下全程自动。
- **角色设定**：支持为两个模型设定具体的专家角色（例如：职业规划专家 + 执行策略专家，或 产品经理 + 工程师）。
- **输出格式**：可根据需求定制输出，例如分步报告、整合建议或最终评分。
- **自动降级**：子代理不可用时自动切换到会话模型切换模式。

## 使用场景
- **复杂问题分析**：需要从多个维度进行深入分析的场景。
- **决策支持**：需要权衡利弊、评估风险的决策过程。
- **代码生成与优化**：一个模型设计架构，另一个模型编写代码或进行代码审查。
- **创意生成**：两个模型互相启发，产生更具创意的想法。
- **跨渠道兼容**：在支持/不支持子代理的渠道间无缝切换。

## 触发命令
- `/duo <模式>`：启动双专家对话，`<模式>` 可以是 `brain_hand` 或 `debate_collaborate`。
- `/duo_auto <模式>`：启动双专家对话，全程自动运行。
- `/duo_step <模式>`：启动双专家对话，每步等待用户确认。
- `/duo_cancel`：取消当前正在进行中的双专家会话。

## 配置
- **默认模型 A (大脑/专家 A)**：`openai/claude-sonnet-4-6`
- **默认模型 B (手/专家 B)**：`dashscope-coding/qwen3.5-plus`
- **默认超时时间**：每步人工确认超时设置为 5 分钟，超时后自动选择默认行为（通常是继续）。
- **降级策略**：`fallback_to_session_switch: true`（默认启用）

## 示例
- `/duo brain_hand "请帮我规划一个 Rust 后端服务的架构，并让 Qwen 生成初步的代码结构。"`
- `/duo_auto debate_collaborate "请两位专家讨论一下 OpenClaw 的长期发展策略，一个从产品角度，一个从技术角度。"`
- `/duo_step debate_collaborate "分析一下我提供的 CSV 数据，从数据质量和业务价值两个角度。"`

---

**注意**：本技能需要模型之间进行多轮交互，可能会产生较高的 Token 消耗，请谨慎使用。在全自动模式下，请确保任务描述清晰，避免意外结果。

**降级模式说明**：在 Webchat 等不支持 `thread=true` 的渠道，技能会自动降级到会话切换模式。虽然功能相同，但两个模型是依次调用而非并行执行，总耗时可能略长。
